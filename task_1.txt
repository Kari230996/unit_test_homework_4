Unit-тесты (семинары)
Урок 4. Зависимости в тестах

---------------------------------------------------------------------------------------------

Задание 1:

1. Использование тестовых заглушек полезно при написании модульных тестов по нескольким причинам. Во-первых, тестовые заглушки позволяют изолировать тестируемый код от зависимостей, таких как внешние системы или базы данных, что делает тесты более надежными и предсказуемыми. Во-вторых, заглушки позволяют сфокусироваться на тестировании конкретного метода или функции без необходимости включать весь окружающий код. Это упрощает отладку и обнаружение проблем в случае сбоев. Также, заглушки могут использоваться для проверки различных сценариев и граничных случаев, что помогает убедиться в правильной работе кода.


2. Для проверки вызова метода с определенными аргументами следует использовать заглушку типа "Mock" или "Spy" (в Mockito). Эти заглушки позволяют проверять, что метод вызывается с определенными аргументами с помощью методов, таких как verify в Mockito.

3. Если вам просто нужно вернуть определенное значение или исключение в ответ на вызов метода, следует использовать заглушку типа "Stub" (в Mockito это делается с помощью метода when(...).thenReturn(...)). Это позволяет настроить ожидаемое поведение заглушки для конкретного вызова метода.

4. Для имитации взаимодействия с внешним API или базой данных следует использовать заглушку типа "Mock" или "Stub". Вы можете настроить их для возвращения предопределенных данных или ожидать определенные вызовы методов без реального взаимодействия с внешней системой.
